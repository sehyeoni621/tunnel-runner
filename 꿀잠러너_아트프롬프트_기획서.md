# 꿀잠 러너 (Tunnel Runner) — 아트 에셋 규격 & 클로드 디자인 프롬프트 기획서

> 게임: HTML5 Canvas 2D 러너 / 아트: 카툰 벡터풍 / 애니메이션: 스프라이트 시트
> 작성일: 2026-07-13

---

## 1. 게임 개요 (프롬프트 컨텍스트용)

- **컨셉**: 지영이 잠을 방해하는 요인들(아빠, 친구, 동생, 밈)을 피해 침대까지 달려가 꿀잠을 자는 러너 게임
- **터널 해석**: 현실 → 꿈나라로 빠져드는 통로. 레벨이 오를수록 어두워지고 별이 많아짐
- **진행**: 레벨제 (1~5) → 클리어 시 코인 → 상점에서 키구루미 잠옷 스킨 구매

---

## 2. 에셋 규격 (Canvas 2D 기준)

### 2-1. 공통 규칙

| 항목 | 규격 |
|---|---|
| 파일 형식 | PNG-24, 배경 투명 |
| 기본 프레임 크기 | 256×256px (게임 내 표시 ~96px, 레티나 대비 여유분) |
| 스프라이트 시트 | 한 시트 최대 1024×1024px, 256px 그리드 정렬 |
| 앵커(기준점) | 발바닥 중앙 하단. 모든 프레임에서 발 위치(y) 동일하게 |
| 여백 | 프레임 가장자리 최소 10% 패딩 (이펙트 잘림 방지) |
| 히트박스 | 스프라이트의 약 70% (코드에서 설정, 아트는 참고만) |
| 파일명 규칙 | `{대상}_{동작}_sheet.png` 예: `jiyoung_base_sheet.png` |

### 2-2. 에셋별 크기

| 에셋 | 프레임 크기 | 프레임 수 | 시트 구성 |
|---|---|---|---|
| 지영 (기본+스킨 각각) | 256×256 | 달리기 6 + 점프 3 + 넘어짐 2 + 잠들기 2 = 13 | 1024×1024 (4×4 그리드) |
| 장애물: 아빠 | 256×384 (세로 큼) | 등장 2 + 대기 2 = 4 | 1024×384 |
| 장애물: 동생(손) | 256×256 | 올라옴 3 | 768×256 |
| 장애물: 조XX / 김XX | 256×256 | 튀어나옴 2 + 대기 1 = 3 (각각) | 768×256 (각각) |
| 장애물: 밈 | 128×128 | 반짝임 2 | 256×128 |
| 코인 | 128×128 | 회전 4 | 512×128 |
| UI 아이콘 (일시정지/설정/상점) | 128×128 | 각 1 | 개별 파일 |
| 버튼 (시작/재시작/구매) | 512×160 | 각 1 | 개별 파일 |
| 상점 스킨 썸네일 | 256×256 | 스킨당 1 (잠들기 포즈 재활용 가능) | 개별 파일 |
| 엔딩 일러스트 (침대+Zzz) | 1024×768 | 1 | 개별 파일 |
| 타이틀 로고 | 1024×512 | 1 | 개별 파일 |

- 배경(우주+별)은 코드로 렌더링 → 이미지 불필요
- 스프라이트 시트는 그리드 어긋나면 코드에서 못 씀 → 생성 후 반드시 256px 격자 확인

## 5. 장애물 프롬프트

각 프롬프트 앞에 [스타일 고정 블록] + "지영 캐릭터와 동일 화풍" 명시.

### 아빠
```
중년 한국 남성 "아빠". 안경, 러닝셔츠+파자마 바지, 팔짱. 못마땅하지만
귀여운 표정. 시트: 1024×384px, 256×384 프레임 4개 —
등장(문 열고 고개 내밈) 2 + 대기(팔짱, 말풍선 "아직 안 자?") 2.
```

### 동생 (바닥 장애물 — 점프 회피)
```
바닥에서 올라오는 어린 동생의 두 손과 정수리. 장난기 가득.
시트: 768×256px, 256×256 프레임 3개 — 그림자만 → 손 반쯤 → 완전히 올라와 잡으려는 포즈.
```

### 친구 조XX (좌우 벽 — 빠른 회피)
```
20대 여성, 스마트폰 든 손과 상반신이 벽에서 튀어나옴. 머리 위에
카톡풍 노란 말풍선 "자니?". 시트: 768×256px, 3프레임 — 튀어나옴 2 + 대기 1.
```

### 친구 김XX (중앙 차선 점거 — 타이밍 회피)
```
20대 여성, 헤드셋 착용, 게임하자고 조르는 포즈(양손 흔들기).
시트: 768×256px, 3프레임 — 좌우로 몸 흔드는 왕복 동작.
```

컨셉: "자기 전에 릴스 하나만 보려다 새벽 3시" — 밈이 지영의 잠을 방해하는
특수 장애물. 시즌마다 유행 밈으로 교체.

## 7. 생성 순서 (권장 워크플로)

1. **지영 기본 시트** → 화풍 확정 (마음에 들 때까지 여기서 반복)
2. 확정본을 레퍼런스로 첨부하고 **스킨 1종 테스트** (곰돌이 추천)
3. 통일성 OK → 나머지 스킨 11종 일괄 생성
4. 장애물 5종 + 코인
5. UI 세트 + 타이틀 로고
6. 엔딩 일러스트
7. 각 시트 256px 그리드 정합성 검수 → 어긋나면 재생성 or 크롭 보정

## 8. 캐릭터 교체 시스템 (추후 이미지 교체 대응)

주인공·아빠·친구·동생 모두 나중에 사용자가 원하는 이미지로 교체 가능하게
하려면, 코드가 특정 그림이 아니라 **"규격+매니페스트"**만 참조하도록 개발 단계에서
아래 구조를 요청할 것.

### 8-1. 폴더 구조

```
assets/
  characters/
    jiyoung/          ← 폴더째 교체 가능
      base_sheet.png
      skins/bear_sheet.png ...
    dad/sheet.png
    friend_jo/sheet.png
    friend_kim/sheet.png
    sibling/sheet.png
  manifest.json       ← 코드는 이 파일만 읽음
```

### 8-2. manifest.json 예시

```json
{
  "jiyoung": {
    "sheet": "characters/jiyoung/base_sheet.png",
    "frameW": 256, "frameH": 256,
    "anims": {
      "run":   { "row": 0, "frames": 6, "fps": 12 },
      "jump":  { "row": 1, "start": 2, "frames": 3 },
      "fall":  { "row": 2, "start": 1, "frames": 2 },
      "sleep": { "row": 3, "frames": 2, "fps": 4 }
    },
    "hitbox": { "scale": 0.7 }
  },
  "dad": { "sheet": "characters/dad/sheet.png", "frameW": 256, "frameH": 384, "...": "..." }
}
```

### 8-3. 교체 규칙 (신규 이미지가 지켜야 할 것)

1. 기존과 동일한 시트 크기·그리드·프레임 수 (2절 규격표 준수)
2. 발 기준선·앵커 동일
3. 파일명 유지 or manifest 경로만 수정
4. 이 3가지만 지키면 코드 수정 없이 즉시 반영

### 8-4. 교체용 이미지 생성 프롬프트 템플릿

사용자가 사진/이미지를 주면 아래 템플릿에 끼워서 생성:
